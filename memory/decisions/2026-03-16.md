# Decisions — 2026-03-16

## Adopted Spec-Driven Development as Standard Workflow
**Context:** Needed a consistent process for turning requirements into implementation
across all projects without cognitive bias or ad-hoc decisions.
**Decision:** Created `/spec` skill (personal, all projects) that enforces
Requirements → Design → Tasks workflow with 5-Question Research Framework.
**Rationale:** Skills ARE specs. The SKILL.md is both spec and implementation.
SDD prevents rework and forces clarity before coding.

**Decision:** Built 7-file system at `~/.claude/`: SOUL.md, IDENTITY.md, USER.md,
MEMORY.md, HEARTBEAT.md, meditations.md + hierarchical memory directories.
**Rationale:** Addresses #1 frustration (re-explaining context) and builds
